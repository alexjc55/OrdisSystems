ФИНАЛЬНАЯ МИГРАЦИЯ VPS СЕРВЕРА
==============================

На основе сравнения схем обнаружено:
- Локальная база данных: 33 колонки слайдера
- VPS база данных: 1 колонка (slider_autoplay)
- НЕДОСТАЕТ: 32 колонки

ВЫПОЛНИТЬ НА VPS СЕРВЕРЕ:

1. Остановить приложение:
   pm2 stop edahouse
   pm2 delete edahouse

2. Скопировать файл миграции на VPS:
   scp complete-slider-migration.sql ordis_co_il_usr@vxaorzmkzo@ordis.co.il:/tmp/

3. Применить полную миграцию:
   psql -h localhost -U edahouse_usr -d edahouse -f /tmp/complete-slider-migration.sql

4. Проверить что все 33 колонки добавлены:
   psql -h localhost -U edahouse_usr -d edahouse -c "SELECT COUNT(*) FROM information_schema.columns WHERE table_name = 'store_settings' AND (column_name LIKE '%slider%' OR column_name LIKE '%slide%');"

   Ожидаемый результат: 33

5. Запустить приложение:
   pm2 start ecosystem.production.config.cjs

6. Проверить что ошибки исчезли:
   pm2 logs edahouse --lines 5

АЛЬТЕРНАТИВНЫЙ СПОСОБ (если нет доступа к файлам):
Выполнить миграцию из EMERGENCY-VPS-STEP-BY-STEP.txt, но добавить все 32 колонки по одной команде.

После миграции все 33 колонки слайдера будут синхронизированы с локальной базой данных.
