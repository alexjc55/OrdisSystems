## Что нам понадобится

Перед началом убедитесь что у вас есть:

- [ ] Аккаунт на [GitHub.com](https://github.com) (бесплатный)
- [ ] Доступ к FastPanel вашего сервера (логин + пароль)
- [ ] IP-адрес сервера
- [ ] SSH логин и пароль (или SSH-ключ) для сервера
- [ ] Домен, который уже указывает на IP сервера (необязательно для начала)

**Программы на вашем компьютере:**
- **Windows**: скачайте [PuTTY](https://putty.org) или используйте встроенный терминал (Win+X → Терминал)
- **Mac/Linux**: используйте встроенный Терминал

## Часть 1 — Отправляем код с Replit на GitHub

> Это нужно делать каждый раз, когда вы меняли что-то в Replit и хотите обновить сайт на сервере.

### Шаг 1.1 — Создаём репозиторий на GitHub (один раз)

1. Зайдите на [github.com](https://github.com) и войдите в аккаунт
2. Нажмите зелёную кнопку **«New»** (или **+** → «New repository»)
3. Заполните:
   - **Repository name**: `НАЗВАНИЕ_САЙТА` (например: `ordis`, `myfood`, `sushishop`)
   - **Private** или **Public** — выберите Private если код должен быть закрытым
4. Нажмите **«Create repository»**
5. Скопируйте адрес репозитория — он будет вида `https://github.com/ВАШ_ЛОГИН/НАЗВАНИЕ_САЙТА.git`

### Шаг 1.2 — Подключаем Replit к GitHub

1. В Replit откройте вкладку **«Version Control»** (иконка ветки в левом меню)
2. Нажмите **«Connect to GitHub»**
3. Разрешите Replit доступ к вашему GitHub аккаунту
4. Выберите созданный репозиторий из списка

### Шаг 1.3 — Отправляем код на GitHub

1. В той же вкладке **«Version Control»**:
   - В поле «Commit message» напишите что изменили, например: `Первая публикация`
   - Нажмите **«Commit & push»**
2. Дождитесь сообщения об успехе ✅

> **Проверка**: зайдите на ваш GitHub репозиторий — там должны появиться файлы проекта.

## Часть 2 — Подключаемся к серверу через SSH

SSH — это защищённое подключение к серверу через командную строку.

### Windows (через встроенный терминал)

1. Нажмите `Win + X` → выберите **«Терминал»** или **«PowerShell»**
2. Введите команду:
```
ssh root@ВАШ_IP_АДРЕС
```
3. Если спросит «Are you sure you want to continue?» — введите `yes` и нажмите Enter
4. Введите пароль (при вводе символы не отображаются — это нормально)

### Mac / Linux

1. Откройте **Терминал** (в Mac: Finder → Программы → Утилиты → Терминал)
2. Введите:
```bash
ssh root@ВАШ_IP_АДРЕС
```
3. Введите пароль

> Если подключение прошло успешно — вы увидите командную строку сервера, например: `root@server:~#`

## Часть 3 — Подготовка сервера (один раз)

> Эти команды выполняются **только один раз** при первой установке.

### Шаг 3.1 — Проверяем и устанавливаем Node.js

```bash
node --version
```

Если видите версию `v18.x.x` или выше — Node.js уже установлен, переходите к шагу 3.2.

Если команда не найдена — установите:
```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
node --version
```

### Шаг 3.2 — Устанавливаем PM2 (менеджер процессов)

PM2 — это программа которая запускает ваш сайт и перезапускает его если он упал.

```bash
npm install -g pm2
pm2 --version
```

### Шаг 3.3 — Устанавливаем Git

```bash
git --version
```

Если не установлен:
```bash
sudo apt-get install -y git
```

### Шаг 3.4 — Создаём папку для сайта

```bash
sudo mkdir -p /var/www/НАЗВАНИЕ_САЙТА
sudo chown $(whoami):$(whoami) /var/www/НАЗВАНИЕ_САЙТА
```

## Часть 4 — Создаём базу данных в FastPanel

### Шаг 4.1 — Входим в FastPanel

1. Откройте в браузере: `https://ВАШ_IP:8888`
2. Введите логин и пароль от FastPanel

### Шаг 4.2 — Создаём базу данных PostgreSQL

1. В левом меню нажмите **«Базы данных»** → **«PostgreSQL»**
2. Нажмите **«Создать базу данных»**
3. Заполните:
   - **Имя базы данных**: `НАЗВАНИЕ_БД`
   - Нажмите **«Создать»**

### Шаг 4.3 — Создаём пользователя базы данных

1. В разделе PostgreSQL найдите **«Пользователи»** → **«Создать пользователя»**
2. Заполните:
   - **Имя пользователя**: `ПОЛЬЗОВАТЕЛЬ_БД`
   - **Пароль**: придумайте надёжный пароль и **запишите его** — он понадобится позже
3. Нажмите **«Создать»**

### Шаг 4.4 — Назначаем права

1. Найдите базу `НАЗВАНИЕ_БД` → нажмите **«Права доступа»**
2. Выберите пользователя `ПОЛЬЗОВАТЕЛЬ_БД`
3. Отметьте все права: SELECT, INSERT, UPDATE, DELETE
4. Нажмите **«Сохранить»**

> **Запишите эти данные — они понадобятся в следующем шаге:**
> - Имя базы: `НАЗВАНИЕ_БД`
> - Пользователь: `ПОЛЬЗОВАТЕЛЬ_БД`
> - Пароль: `ВАШ_ПАРОЛЬ`
> - Хост: `localhost`
> - Порт: `5432`

## Часть 6 — Создаём файл настроек .env

Файл `.env` содержит все секретные настройки сайта. Создаём его вручную:

```bash
nano /var/www/НАЗВАНИЕ_САЙТА/.env
```

Откроется текстовый редактор. Вставьте следующее содержимое (замените значения на свои):

```env
# Режим работы (не менять)
NODE_ENV=production

# Порт (не менять)
PORT=3000

# База данных PostgreSQL
DATABASE_URL=postgresql://ПОЛЬЗОВАТЕЛЬ_БД:ВАШ_ПАРОЛЬ@localhost:5432/НАЗВАНИЕ_БД
PGHOST=localhost
PGPORT=5432
PGDATABASE=НАЗВАНИЕ_БД
PGUSER=ПОЛЬЗОВАТЕЛЬ_БД
PGPASSWORD=ВАШ_ПАРОЛЬ

# Секретный ключ сессий — придумайте длинный случайный текст (мин. 32 символа)
SESSION_SECRET=НаберитеЗдесьЛюбойДлинныйСлучайныйТекстКоторыйНиктоНеЗнает1234567890

# Отправка email через SendGrid (необязательно — если не нужна почта, оставьте пустым)
SENDGRID_API_KEY=

# Push-уведомления (необязательно — если не нужны, оставьте пустыми)
VAPID_PUBLIC_KEY=
VAPID_PRIVATE_KEY=
VAPID_EMAIL=

# Не использовать Neon DB (важно для VPS)
USE_NEON=false
```

**Как сохранить файл в nano:**
- Нажмите `Ctrl + X`
- Нажмите `Y` (подтвердить сохранение)
- Нажмите `Enter`

### Как создать SESSION_SECRET

Выполните команду — она создаст случайный ключ:
```bash
openssl rand -base64 48
```

Скопируйте результат и вставьте вместо `НаберитеЗдесьЛюбойДлинныйСлучайныйТекстКоторыйНиктоНеЗнает1234567890`

## Часть 8 — Переносим базу данных

Здесь два варианта в зависимости от ситуации.

---

### Вариант А — Свежая установка (база пустая, заполним из Replit)

#### Шаг А1 — Экспортируем данные из Replit

На вашем **компьютере** (не на сервере) — в Replit откройте Shell и выполните:

```bash
pg_dump "$DATABASE_URL" \
  --no-owner \
  --no-acl \
  --format=plain \
  --file=database_export.sql
```

Если команда не работает попробуйте:
```bash
pg_dump "$DATABASE_URL" -Fp --no-owner --no-acl > database_export.sql
```

#### Шаг А2 — Скачиваем файл с Replit

В Replit в левом меню найдите файл `database_export.sql`, кликните правой кнопкой → **«Download»**.

#### Шаг А3 — Загружаем файл на сервер

**Для Windows** (в PowerShell):
```powershell
scp C:\путь\к\database_export.sql root@ВАШ_IP:/var/www/НАЗВАНИЕ_САЙТА/
```

**Для Mac/Linux** (в Терминале):
```bash
scp ~/Downloads/database_export.sql root@ВАШ_IP:/var/www/НАЗВАНИЕ_САЙТА/
```

#### Шаг А4 — Импортируем в PostgreSQL на сервере

Возвращаемся в SSH и выполняем:
```bash
psql postgresql://ПОЛЬЗОВАТЕЛЬ_БД:ВАШ_ПАРОЛЬ@localhost:5432/НАЗВАНИЕ_БД < /var/www/НАЗВАНИЕ_САЙТА/database_export.sql
```

> Появятся сообщения о создании таблиц и добавлении данных — это нормально.

### Вариант Б — Обновление (база уже есть, применяем миграции)

Если сайт уже работал и нужно только применить новые изменения схемы:

```bash
cd /var/www/НАЗВАНИЕ_САЙТА

npm run db:push

# Или применяем конкретный SQL-файл миграции (если есть)
```

## Часть 9 — Запускаем сайт

### Шаг 9.1 — Запуск через PM2

```bash
cd /var/www/НАЗВАНИЕ_САЙТА
pm2 start dist/index.js --name НАЗВАНИЕ_САЙТА
```

> Замените `НАЗВАНИЕ_САЙТА` на имя вашего проекта, например `ordis` или `myfood`.

### Шаг 9.2 — Проверяем что сайт запустился

```bash
pm2 status
```

Должны увидеть строку с вашим названием сайта и статусом `online` (зелёный).

```bash
pm2 logs НАЗВАНИЕ_САЙТА --lines 20
```

Проверяем логи — ищем строку вида `serving on port 3000`. Ошибок не должно быть.

Быстрая проверка что сайт отвечает:
```bash
curl http://localhost:3000/api/health
```

Должны увидеть: `{"status":"healthy",...}`

### Шаг 9.3 — Включаем автозапуск PM2 при перезагрузке сервера

```bash
pm2 startup
```

Эта команда выведет одну команду — скопируйте и выполните её.

Затем:
```bash
pm2 save
```

## Часть 10 — Настраиваем Nginx в FastPanel

Nginx — это «посредник» который принимает запросы из интернета (порт 80/443) и перенаправляет их на ваш Node.js сайт (порт 3000).

### Шаг 10.1 — Создаём сайт в FastPanel

1. В FastPanel перейдите в **«Сайты»** → **«Создать сайт»**
2. Заполните:
   - **Домен**: `ваш-домен.com`
   - **Тип**: PHP или Static — **неважно**, мы заменим конфиг
   - **Директория**: `/var/www/НАЗВАНИЕ_САЙТА`

### Шаг 10.2 — Настраиваем конфигурацию Nginx

1. В FastPanel найдите сайт → **«Настройки»** → **«Nginx конфиг»** или **«Дополнительная конфигурация»**
2. Замените содержимое на:

```nginx
server {
    listen 80;
    server_name ваш-домен.com www.ваш-домен.com;

    # Максимальный размер загружаемых файлов
    client_max_body_size 50M;

    # Загруженные изображения отдаём напрямую
    location /uploads/ {
        alias /var/www/НАЗВАНИЕ_САЙТА/uploads/;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # Всё остальное — перенаправляем на Node.js
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        proxy_read_timeout 300;
    }
}
```

3. Сохраните и нажмите **«Применить»** или **«Перезапустить Nginx»**

### Шаг 10.3 — Устанавливаем SSL сертификат (HTTPS)

В FastPanel найдите ваш сайт → **«SSL»** → **«Let's Encrypt»** → **«Выпустить сертификат»**

После выпуска сертификата в конфиге Nginx автоматически добавится секция с HTTPS.

1. Откройте браузер и зайдите на ваш домен: `https://ваш-домен.com`

**Проверочный список:**

- [ ] Товары отображаются (значит база работает)
- [ ] Можно войти в admin-панель: `https://ваш-домен.com` → войти как admin
- [ ] Изображения товаров загружаются (значит папка uploads настроена)

**Если что-то не работает** — смотрите раздел [Решение проблем](#решение-проблем).

## Решение проблем

### Сайт не открывается в браузере

```bash
# Проверяем что PM2 запущен
pm2 status

# Смотрим логи ошибок
pm2 logs НАЗВАНИЕ_САЙТА --lines 50

# Проверяем что порт 3000 слушается
ss -tlnp | grep 3000
```

### Ошибка «Cannot connect to database»

```bash
# Проверяем подключение к PostgreSQL
psql postgresql://ПОЛЬЗОВАТЕЛЬ_БД:ПАРОЛЬ@localhost:5432/НАЗВАНИЕ_БД -c "SELECT 1;"
```

Если ошибка — проверьте:
1. Правильность пароля в файле `.env`
2. Что PostgreSQL запущен в FastPanel

### Ошибка 502 Bad Gateway в браузере

Node.js сайт не запущен. Выполните:
```bash
pm2 start dist/index.js --name НАЗВАНИЕ_САЙТА
```

### Не загружаются изображения

```bash
# Проверяем папку uploads
ls -la /var/www/НАЗВАНИЕ_САЙТА/uploads/

# Устанавливаем права
chmod 755 /var/www/НАЗВАНИЕ_САЙТА/uploads
```

### Смотрим полные логи ошибок

```bash
# Логи Node.js приложения
pm2 logs НАЗВАНИЕ_САЙТА

# Логи Nginx (искать в FastPanel или:)
tail -f /var/log/nginx/error.log
```

### Перезапуск с нуля

Если что-то пошло совсем не так:
```bash
pm2 stop НАЗВАНИЕ_САЙТА
pm2 delete НАЗВАНИЕ_САЙТА
cd /var/www/НАЗВАНИЕ_САЙТА
pm2 start dist/index.js --name НАЗВАНИЕ_САЙТА
```
