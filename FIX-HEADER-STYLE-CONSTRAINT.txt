ИСПРАВЛЕНИЕ ОШИБКИ HEADER_STYLE CONSTRAINT
=========================================

ОШИБКА:
constraint: 'store_settings_header_style_check' - означает что значение header_style не соответствует разрешенным в базе данных.

КОМАНДЫ ДЛЯ ИСПРАВЛЕНИЯ:

1. Проверить текущие ограничения для header_style:
psql -h localhost -U edahouse_usr -d edahouse -c "SELECT conname, pg_get_constraintdef(oid) FROM pg_constraint WHERE conname LIKE '%header_style%';"

2. Проверить текущее значение header_style:
psql -h localhost -U edahouse_usr -d edahouse -c "SELECT header_style FROM store_settings WHERE id = 1;"

3. Удалить неправильное ограничение:
psql -h localhost -U edahouse_usr -d edahouse -c "ALTER TABLE store_settings DROP CONSTRAINT IF EXISTS store_settings_header_style_check;"

4. Установить правильное значение по умолчанию:
psql -h localhost -U edahouse_usr -d edahouse -c "UPDATE store_settings SET header_style = 'default' WHERE id = 1;"

5. Перезапустить приложение:
pm2 restart edahouse

6. Проверить логи:
pm2 logs edahouse --lines 5

РЕЗУЛЬТАТ: Ошибка constraint исчезнет и приложение будет работать без ошибок.
